// git 第一天 // 主要做代码托管 // 版本控制软件：用来记录文件的变化（新增文件、修改文件、删除文件等等操作；用于团队协作） // 版本控制的分类：本地 -- 集中化 -- 分布式（比较主流，代表工具：git） // 集中化 -- SVN 缺点：依赖网络(只记录变化的部分，可以节省磁盘空间) // 分布式 -- Git 优点：在没有网络的状态，依旧可以本地提交（不仅仅是记录变化的部分，把原来的也拷贝了一份，占用磁盘空间大，但版本切换时非常快 空间换时间 // 特点：git所有的操作近乎都是在本地运行的（速度会很快） // 三个区域：工作区 暂存区 仓库 // 三个状态：已修改modified 已暂存staged 已提交committed // 流程：对文件进行修改->需要进行暂存->总的提交（本地提交，提交到仓库） // shift键+鼠标右键 可以进入powershell // git配置用户名和邮箱 // git config --global user.name "名字" // git config --global user.email "邮箱" // git查看用户名和邮箱 // git config user.name // git config user.email // 全局配置的用户名和邮箱具体配置的位置 C:\Users\用户名\.gitconfig //查看全局所有的配置项：git config --list --global // 获取帮助信息 git help config // git config -h // 使用 cls 可以清除之前输入的命令 清屏 // 获取git仓库的两种方式 // 1、可以将本地的某个文件夹转换成git仓库（需要执行git init命令 ） 不要在桌面弄 // 2、可以直接从远程服务器克隆一份下来 // 注意点：新建的工程不要有中文，名称中间不要有空格等特殊字符 // 文件所处的状态： // 1.未跟踪的 untracked(当前文件还没有被git所管理 新建的文件) // 2.未修改的 unmodified // 3.已修改 modified // 4.已暂存 staged // 查看文件的状态：git status 文件处于什么状态 都能看到 // 精简的命令：git status -s ??标识当前文件处于未被跟踪的状态 // 跟踪新文件：git add 你要跟踪的文件的名称(可以使用tab键，快速补齐路径)(使用上下左右的上键，可以快速恢复到上一次输入的命令) // 提交更新文件:git commit -m '此次提交的记录信息' (-m 后面的提交信息，一定不能丢) 哈哈哈哈哈哈哈哈哈哈哈ggggggggggggggggggggggggggggggg
